Introduction

Data Type - defines a collection of data values and a set of predefined operations on those values.

  • Computer programs produce results by manipulating data.
  • Early languages used only a few basic language-supported data structures.
  • Advances in Data Type Design
    • User-Defined Types
      • Introduced in ALGOL 68 - one of the most important advancements in the evolution of data type design
    • Abstract Data Types
      • Interface (visible) of a type is separate from the implementation (hidden) of that type
      • All of the types provided by a high-level language are abstract data types
  • Type System: a set of types and the rules that govern their use in programs
    • Benefits
      • Error Detection through Type Checking
      • Program Modularization through cross-module Type Checking that ensures the consistency of the interfaces among the modules
      • Documentation - type declarations provide clues about a programs behavior
  • Descriptor - Convenient way to think of variables - collection of attributes (name, type, address, value, lifetime, scope) of a variable
    • If attributes are static, only needed at compile time
    • For Dynamic attributes, part or all of the descriptor is maintained during execution
