Primitives

  • Primitive data types: Those not defined in terms of other types
    • Provided by most all programming languages
    • Some merely reflections on hardware
    • Others need only minimal hardware support for implementatioin
  • Integer:
    • Up to eight integer types in a language
    • Java has byte, short, int, and long
  • Floating Point:
    • Approximations of real numbers (pi and e cannot be correctly represented)
    • IEEE Floating-Point Standard 754, represented as fractions and exponents.
    • The numerical value of a finite number is  (−1)s × c × bq where b is the base (2 or 10). For example, if the sign is 1 (indicating negative), the significand is 12345, the exponent is −3, and the base is 10, then the value of the number is −12.345.

  • Complex:
    • C99, Fortran, Python
    • Ordered pairs of floating point values (e.g. - In Python - (7 + 3j), where j follows the imaginery part)
  • Decimal:
    • Used for money
    • Store a fixed number of decimal digits in Binary Coded Decimal (BCD) form
    • Advantage: accuracy
    • Disadvantages: limited range, wastes memory
  • Boolean:
    • "true" or "false"
    • Implemented as bytes
    • Advantage: readability
  • Character:
    • Stored as numeric codings (ASCII, UCS-2, UCS-4)
